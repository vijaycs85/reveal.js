New routing system

hook_menu

We (will) miss you :(


                        /**
                        * Implements hook_menu().
                        */
                        function user_menu() {
                        $items['admin/config/people/accounts'] = array(
                        'title' => 'Account settings',
                        'description' => 'Configure default behavior of users, including registration requirements, e-mails, and fields.',
                        'weight' => -10,
                        'route_name' => 'user.account_settings',
                        );
                        return $items;
                        }

hook_menu part 2


    // user.routing.yml
    user.account_settings:
    path: '/admin/config/people/accounts'
    defaults:
    _form: '\Drupal\user\AccountSettingsForm'
    requirements:
    _permission: 'administer account settings'

Configuration system

No more Variables

  1. config
  2. state
  3. settings

Config

Anything that need to be moved

  1. config('')->get()
  2. config('')->set()->save()
  3. config('')->del()

State

Anything that's specific to the installation

Settings

Anything that's specific to environment or non-configurable

Web services

Build-in web services

  • Expose content as JSON or XML.

  • HTTP authentication.

  • Expose Views-generated lists as services.

Fields, Views

Fields

  • Fields are saved in config files.

  • More field type in core: entity reference, link, date, e-mail, telephone, etc.

  • Comments are now a field

  • Add fields to blocks

Views

  • Views in core!!! (incl. VBO)

  • All list in core are/will be views (front page, block list page, etc.)

Mobile First

  • Responsive themes

  • Mobile content administration
